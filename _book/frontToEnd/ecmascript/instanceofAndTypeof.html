JS中typeof与instanceof之间的区别总结

typeof 与instanceof 都是常用来判断一个变量是不是为空或者是什么类型，但是两个还是有区别的。

typeof 是一个一元运算符，但是优先级比较高。返回值有以下几种:

1. console.log(typeof 12); //"number"
2. console.log(typeof "123") //"string"
3. console.log(typeof true) //"boolean"   
4.console.log(typeof [1,2]) //"object"
5.console.log(typeof {name:123}) //object
6.console.log(typeof null); //object
7.console.log(typeof undefined); //undefined
8.console.log(typeof function(){}); //function
number , string , boolean , object , undefined , function

instanceof运算符可以用来判断某个构造函数的prototype属性是否存在于另外一个要检测对象的原型链上。

function C(){};
function D(){};


var o = new C();

console.log(o instanceof C);  // true 
console.log(o instanceof D); // false


C.prototype = {};
console.log(o instanceof C); // false  这里是因为C的原型指向空对象，可以理解成指针对象指向空的，所以o是不在C的原型链里面
